## OVERVIEW

The ZIGGIE project utilizes a three-tier hierarchical agent system with **1,884 total agents** working in concert to create game assets, manage workflows, and coordinate development.

### Architecture Summary

- **Total Agents:** 1,884
- **L1 (Strategic) Agents:** 12
- **L2 (Tactical) Agents:** 144 (12 per L1)
- **L3 (Micro) Agents:** 1,728 (12 per L2, 144 per L1)

### Mathematical Structure

```
12 L1 × 12 L2 × 12 L3 = 1,884 Total Agents
```

## AGENT NAMING CONVENTION

### Format
```
LX.Y.Z
```

Where:
- **X** = Tier level (1, 2, or 3)
- **Y** = Parent agent number (for L2/L3)
- **Z** = Agent number within parent (for L3)

### Examples
- `L1.3` = Environment Pipeline Agent (L1 tier, 3rd agent)
- `L2.3.7` = L1.3's 7th L2 agent (Integration specialist)
- `L3.3.7.12` = L2.3.7's 12th L3 micro-agent (Knowledge synthesizer)

## COMMUNICATION PATTERNS

### Vertical Communication
- **L1 → L2:** Strategic directives and goals
- **L2 → L3:** Tactical tasks and assignments
- **L3 → L2:** Task completion and results
- **L2 → L1:** Progress reports and issues

### Horizontal Communication
- **L1 ↔ L1:** Cross-domain coordination
- **L2 ↔ L2:** Pipeline integration
- **L3 ↔ L3:** Resource sharing (within same L2)

### Cross-Tier Communication
- **L1 ↔ L3:** Emergency escalation only
- **All Tiers:** Knowledge base access
