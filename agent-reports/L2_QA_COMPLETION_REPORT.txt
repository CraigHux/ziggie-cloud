- Zero JavaScript errors in console
- Authentication token sent correctly
- WebSocket connects successfully

Backend:
- All endpoints respond with correct status codes
- Return real metrics (0-100%, not hardcoded)
- JWT tokens valid and formatted correctly
- Database connected and responsive
- Error messages include details (not generic 500)

Integration:
- Frontend can reach backend (CORS working)
- Data flows end-to-end correctly
- Errors handled gracefully (no crashes)
- Performance acceptable (less than 500ms API, less than 3s page)

PREVENTION - What MUST Be Verified Before Sign-Off
================================================================================

MANDATORY CHECKS (13 items):

1. Real metrics display
   - CPU percent between 0-100 (not 0.0%)
   - Memory and Disk same rules

2. Frontend-Backend connection
   - CORS working (no cross-origin errors)
   - No 401 or 403 authentication failures

3. Zero console errors
   - F12 to Console tab
   - RED errors equals 0 (yellow warnings acceptable)

4. Valid API responses
   - All responses are valid JSON (not HTML error)
   - No hardcoded placeholder values

5. Authentication working
   - Login returns real JWT (not "token123")
   - Bearer token sent with requests

6. Database ready
   - Database file exists and accessible
   - Tables initialized (if needed)

7. No mock values
   - All displayed values are real
   - No hardcoded timestamps

8. Error responses useful
   - Include message field explaining problem
   - No SQL errors or stack traces exposed

9. No sensitive data leaks
   - Error messages don't expose passwords or keys
   - Database paths not shown to users

10. Graceful failure handling
    - Backend offline shows clear error message
    - Frontend doesn't crash

11. WebSocket functional
    - Connection establishes
    - Real-time updates flow

12. Performance acceptable
    - Page load less than 3 seconds
    - API response less than 500ms

13. No orphaned processes
    - Backend shutdown clean
    - No lingering service processes
