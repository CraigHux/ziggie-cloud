## Search Tips

### Finding Files

**By Name:**
```bash
# Find specific file
find . -name "README.md"

# Find by pattern
find . -name "*API*.md"
```

**By Content:**
```bash
# Search in markdown files
grep -r "keyword" --include="*.md"

# Search in Python files
grep -r "keyword" --include="*.py"
```

## File Naming Conventions

### Markdown Files
- `UPPERCASE.md` - Important top-level docs (README, ARCHITECTURE)
- `Title_Case.md` - Standard documentation
- `lowercase.md` - Component-specific files

### Code Files
- `snake_case.py` - Python files
- `PascalCase.tsx` - React components
- `camelCase.ts` - TypeScript utilities

### Configuration Files
- `.lowercase` - Dotfiles (.gitignore, .env)
- `lowercase.json` - JSON configs
- `lowercase.yaml` - YAML configs

## Documentation Standards

### All Docs Should Have:
- Clear title
- Table of contents (if >500 lines)
- Version/date information
- Purpose statement
- Examples where applicable

### Markdown Guidelines:
- Use headers (#, ##, ###)
- Code blocks with language tags
- Tables for comparisons
- Links to related docs
- Clear formatting
